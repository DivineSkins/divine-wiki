**LtMAO** is one of the most essential tools in the League of Legends modding community. It provides a convenient right-click context menu for quick file conversions and operations, making it indispensable for any modder's workflow.

## Key Features

LtMAO's right-click menu allows you to:

### File Conversions

| From   | To     | Use Case                                       |
| ------ | ------ | ---------------------------------------------- |
| `.dds` | `.tex` | Convert textures for use in-game               |
| `.tex` | `.dds` | Extract textures for editing                   |
| `.wem` | `.wav` | Convert audio files to editable format         |
| `.bin` | `.py`  | Convert binary files to readable Python format |
| `.py`  | `.bin` | Repack edited Python files back to binary      |

### WAD Operations

- **Unpack `.wad.client` files** with custom extracted hashes
- Essential for working with **repathed** or **hard-modified** mods
- Extract individual files or entire WAD archives

### File Information

Print detailed information about:

- **`.bin` files** – View structure and properties
- **`.skn` files** – Skin mesh information
- **`.skl` files** – Skeleton/rig data

## Installation

1. Download: [LtMAO-hai.zip](https://github.com/tarngaina/LtMAO/archive/refs/heads/hai.zip)
2. Extract: `LtMAO-hai.zip`
3. Run `LtMAO-hai/start.bat`

## Updates

1. LtMAO-hai always checks for the latest version when you open the app. If there is a newer version, the app will display an update message in the title bar.
2. To fully update: re-download the whole app from GitHub and extract it to the same location.
3. To lazily update LtMAO: press "Redownload LtMAO" from the settings page. Not recommended, but if it works, it works.

### Theme

You can create your custom theme by making a new folder in `LtMAO-hai/res/themes`.

Your custom image sizes must be the same ratio as the original images.

The accent color of the app will be the dominant color of your `background.gif`.

If there is `_` in the theme name, the accent color will be a gradient between the two most dominant colors.

### hash_helper

<img
  src="https://i.postimg.cc/7Y0W2XX0/screenshot-1.png"
  alt="hash_helper interface"
  width={700}
/>

**Important:** wait for all syncing, updating, and hash loading to finish before running any `LtMAO` functions.

1. CDTB Hashes: auto-syncs [CommunityDragon](https://github.com/CommunityDragon/CDTB/tree/master/cdragontoolbox) hashes. Can also be manually downloaded at the mentioned link.
2. Extracted Hashes: extracted personally by the user. Hashes that can be extracted:
   - binentries:
     - VfxSystemDefinitionData → particlePath in BIN.
     - StaticMaterialDef → name in BIN.
   - binhashes:
     - Joint hashes → joint names in SKL.
     - Submesh hashes → submesh names in SKN.
   - game:
     - File paths that start with `assets/` or `data/` in BIN. If the file type is `.dds`, 2x and 4x dds are extracted too.
3. Custom Hashes:
   - Custom Hashes is the hash set used by all LtMAO-related functions: ritobin, wad_tool, no_skin, etc.
   - Custom Hashes = CDTB Hashes + Extracted Hashes + User Manually Added Hashes

`hash_helper` also has a wad & bin hash generator. Those generated hashes can be added to Custom Hashes with a button.

### no_skin

Create a NO SKIN mod: replaces **almost** every League skin with the default.

<img
  src="https://i.imgur.com/AfQyzFN.png"
  alt="no_skin tool interface"
  width={700}
/>

`SKIPS.json`: some skins crash League when swapped to base. This file tells the program not to change those skins to base.

Has two modes:

1. Full: makes a full `NO SKIN.fantome` of all champions inside `League of Legends/Game/DATA/FINAL/Champions`.

   <img
     src="https://i.postimg.cc/ht0ZCjkG/screenshot-4.png"
     alt="no_skin Full mode output"
     width={700}
   />

2. Lite: makes all selected `skinx.bin` become `skin0.bin`.
